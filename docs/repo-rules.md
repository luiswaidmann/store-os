# Repo Rules

1. Alles, was für mehrere Projekte gelten soll, bleibt generisch.

2. In diesen Ordnern darf nichts SuppliedTech-Spezifisches stehen:
- core
- schemas
- prompts
- workflows
- validators
- shared

3. Alles, was nur für ein einziges Projekt gilt, kommt nur in:
- projects/<project-id>/

4. Alte Themes sind nur Referenz, nicht die Wahrheit.

5. Marktanalyse kommt vor Design.

6. Wettbewerber dürfen nicht direkt kopiert werden.

7. Live-Publish nur nach Prüfung.
